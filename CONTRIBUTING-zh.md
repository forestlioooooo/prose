# 贡献 OpenProse

感谢您有兴趣为 OpenProse 做出贡献！本文档概述了如何参与。

## 贡献方式

- **报告错误** — 打开一个问题描述出错情况
- **建议功能** — 打开一个问题分享您的想法
- **提交 PR** — 修复错误或实现功能
- **分享反馈** — 告诉我们您如何使用 OpenProse

## 开始之前

### 小改动

对于错误修复、文档改进或小增强功能 — 直接打开 PR。无需事先询问。

### 重大改动

在对主要功能或架构变更投入大量时间之前，请先联系我们。这有助于确保您的工作与项目方向一致，并且不会与正在进行的开发冲突。

讨论想法的方式：
- 在 GitHub 上打开一个问题描述您的提案
- 在 X/Twitter 上联系：[@irl_danB](https://x.com/irl_danB)

这不是设门槛 — 而是为了避免您的工作与已在进行的内容冲突而浪费精力。

## 提交 PR

1. Fork 仓库
2. 为您的变更创建分支
3. 进行更改
4. 本地测试（运行示例 `.prose` 程序）
5. 提交带有清晰描述的 PR

## 行为准则

保持尊重和建设性。我们正在构建新颖和实验性的东西 — 深思熟虑的反馈对大家都有帮助。
